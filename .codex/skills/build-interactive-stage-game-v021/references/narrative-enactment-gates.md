# 0.21 Narrative Enactment Gates

Adapt the script as a chain of witnessed events, not as a list of facts delivered by NPCs.

## Build the source-event ledger

Create `generated/30-performance/source-event-ledger.json` before approving the performance plan. Preserve source order for every included event cluster and record the exact source span. Classify each cluster as `arrival`, `discovery`, `transformation`, `confrontation`, `ritual`, `departure`, or `other`.

Every included cluster declares:

- `transitionIn` and the prior condition that makes the event intelligible;
- ordered `phases`, each with actors, player witness, delivery mode, dialogue intent, physical action, and world effects;
- `transitionOut`, visible consequence, and the next dramatic question;
- omitted source material and the canon-safe reason for omission.

Use these minimum phase sets:

| Sequence | Required phases |
| --- | --- |
| arrival | setup, signal, response, threshold-action, introduction, orientation, consequence |
| discovery | setup, notice, inspect, interpretation, consequence |
| transformation | foreshadow, trigger, visible-change, reaction, new-affordance |
| confrontation | approach, claim, resistance, turn, consequence, breath |
| ritual | preparation, placement, response, escalation, revelation, aftermath |
| departure | decision, preparation, threshold-action, separation, aftermath |
| other | setup, event, reaction, consequence, transition |

## Enact before explaining

- Deliver physical and social events as `enacted` or `interactive`. Do not mark them `reported-only`.
- Show a visitor signal before a door opens; show the threshold action before introductions; orient newcomers before they begin speaking as established residents.
- Let actors enter, cross, handle a prop, look, wait, interrupt, or react before dialogue explains meaning.
- Use dialogue to establish identity, intention, relationship, uncertainty, or subtext. Do not compress an event into a cryptic hint whose setup was never shown.
- Give every location change a bridge: intention, movement cue, threshold, arrival image, and renewed local objective.
- After a major reveal or transformation, include a reaction and breath before the next mandatory objective.

## Block approval

Reject the performance plan when source-order changes are unexplained, a required phase is absent, a physical event exists only in dialogue, an NPC speaks as if an unshown introduction already happened, or more than two consecutive required phases are dialogue-only.
