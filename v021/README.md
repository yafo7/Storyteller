# 雾宅：另一扇门（Skill 0.21）

这是由 `build-interactive-stage-game-v021` 独立生成的俯视角像素叙事游戏。它使用《小岛惊魂》本地剧本，但不调用 0.3 的游戏设计库，目的是形成可与 0.2、0.3 直接比较的像素 Profile 基线。

启动仓库根目录的服务器：

```powershell
npm run dev
```

浏览器打开：<http://127.0.0.1:5175/v021/>

操作：

- `WASD` / 方向键：四向移动
- `E` / 空格：面对目标互动、推进对话
- `J`：目标、提示、证物与事实日志
- `Esc`：菜单、保存、重开与结局重播
- `F2`：开发调试层（碰撞、传送点、互动热点）
- 触屏设备：使用屏幕方向键和互动键

完整主路线：育儿室关帘 → 门厅未寄信 → 育儿室 Victor 层 → 音乐室调谐与肖像册 → 白昼大厅 → 花园三墓碑 → 降灵桌三证物 → 育儿室晨光。

验证：

```powershell
npm run check:v021
```

生成产物、路线报告、验收报告与浏览器截图分别位于 `generated/` 和 `reports/`。存档键为 `storyteller-v021-save`，不会覆盖 0.1 或 0.2 的进度。

## 0.21-R2 美术生产实验

R2 不再把风格图停留在参考阶段。工作流先冻结 `style-contract.json`，再分别保存六名角色身份与立绘源文件、五张场景底图、对齐的地图状态变体和九种关键道具状态，最后才编译运行时图集。标题页和五张地图直接加载这些正式美术资产。

`asset-coverage.json` 负责检查人物、场景和道具是否完整；`visual-validation.json` 将风格母版、独立源文件、运行时资产和 Edge 截图连接起来。任何正式资产包含 `placeholder`、缺少上游引用或没有引擎截图时，严格校验都会失败。
