# Scope policy 1.0.0

`zelda-mainline` uses a reproducible project definition, not an unsupported claim that Nintendo officially uses the word “mainline”. A work is included when Nintendo's Zelda Portal HISTORY presents it as a series-history game node. When one node is officially documented as multiple content-distinct games, each game receives a separate `workId`.

The frozen 2026-07-17 snapshot has 20 Portal nodes and 21 works. *Oracle of Ages* and *Oracle of Seasons* are separate works in one paired release family. Multiplayer-focused Portal entries remain included. Product-list spin-offs, guest appearances, compilations, emulated rereleases, hardware bundles, and adjacent Warriors/Cadence products are boundary records, not corpus works.

The Portal's dates are stored explicitly as Japanese catalogue dates. They must never be relabelled as worldwide first-release dates. Regional events, concurrent original platforms, ports, remasters, remakes, enhanced editions, DLC, and unknown future editions remain versions of the same work. A version whose nature is not officially described stays `review`; it does not create a new work or block work-level scope resolution.

Scope releases are immutable. New official evidence creates a new semantic scope version and a migration record. Frozen scope requires authority-union reconciliation of 100%, every candidate resolved to included/excluded, and zero work-level `review` decisions.
